Tipe Testing Kategori Definisi Tujuan Utama Contoh Konkret
01
Unit Testing
Unit Test
 Functional Pengujian pada unit terkecil kode (fungsi, method, atau class) secara terisolasi dari komponen lain. Memastikan setiap unit kode bekerja sesuai logikanya sendiri.
Contoh
Fungsi hitungDiskon(harga, persen) ditest dengan input harga=100.000 dan persen=20 — expected output: 80.000.
02
Integration Testing
Integration Test
 Functional Pengujian interaksi antara dua atau lebih modul/komponen yang digabungkan. Memastikan modul-modul bekerja dengan benar saat dihubungkan satu sama lain.
Contoh
Modul Login dihubungkan dengan modul Database — test memverifikasi bahwa data user berhasil diambil dari DB saat login.
03
System Testing
System Test
 Functional Pengujian pada keseluruhan sistem yang sudah terintegrasi sebagai satu kesatuan. Memastikan sistem memenuhi semua functional requirement yang telah ditentukan.
Contoh
User melakukan alur lengkap: registrasi → login → tambah produk ke cart → checkout → pembayaran — semua proses diverifikasi end-to-end.
04
Acceptance Testing
UAT / Acceptance
 Functional Pengujian oleh user atau client untuk memvalidasi apakah sistem memenuhi kebutuhan bisnis mereka. Mendapatkan persetujuan user/client bahwa sistem siap digunakan di production.
Contoh
Tim finance client menguji fitur laporan keuangan bulanan — mereka memvalidasi apakah data dan format laporan sesuai kebutuhan mereka.
05
Smoke Testing
Build Verification
 Functional Pengujian cepat untuk memverifikasi apakah build terbaru stabil dan layak untuk dilanjutkan ke testing yang lebih dalam. Mendeteksi masalah kritis sejak awal sebelum membuang waktu untuk full testing.
Contoh
Setelah build baru dirilis, QA mengecek: apakah aplikasi bisa dibuka? apakah login berfungsi? apakah halaman utama bisa diakses? — jika gagal, build dikembalikan ke developer.
06
Sanity Testing
Sanity Check
 Functional Pengujian cepat untuk memverifikasi bahwa bug tertentu sudah difix atau fitur baru bekerja sesuai ekspektasi. Memastikan perubahan spesifik sudah benar sebelum melakukan full regression.
Contoh
Developer fix bug "harga tidak tampil di halaman detail produk" — QA langsung mengecek halaman tersebut untuk memverifikasi fix sudah benar.
07
Functional Testing
Feature Test
 Functional Pengujian yang memverifikasi apakah setiap fitur sistem bekerja sesuai dengan spesifikasi requirement. Memastikan semua fungsi yang dijanjikan dalam requirement dapat berjalan dengan benar.
Contoh
Fitur filter produk berdasarkan kategori ditest — QA memilih "Elektronik" dan memverifikasi bahwa hanya produk elektronik yang ditampilkan di hasil pencarian.
08
API Testing
API Test
 Functional Pengujian langsung pada API (endpoint, request/response, status code, dan data) tanpa melalui UI. Memastikan API mengembalikan data yang benar, response code yang tepat, dan menangani error dengan baik.
Contoh
POST /api/orders dikirim dengan body yang valid — QA memverifikasi response 201 Created, ID order terbuat, dan data order tersimpan di database.
09
UI Testing
Interface Test
 Functional Pengujian pada antarmuka pengguna — mencakup tampilan, interaksi, dan navigasi di browser atau aplikasi. Memastikan UI berfungsi dengan benar, responsif, dan sesuai dengan desain yang disepakati.
Contoh
QA memverifikasi bahwa form pendaftaran: label terbaca jelas, validasi muncul saat input kosong, dan tombol Submit aktif hanya setelah semua field diisi.
10
Exploratory Testing
Ad-hoc Explore
 Functional Pengujian tanpa test case yang sudah ditentukan sebelumnya — QA bebas mengeksplorasi sistem menggunakan intuisi dan pengalaman. Menemukan bug yang tidak terpikir dalam test case formal, terutama edge case dan unexpected behavior.
Contoh
QA mencoba login dengan karakter emoji di field password, copy-paste teks sangat panjang, dan double-click tombol Submit secara cepat — ditemukan bug: double-click menghasilkan 2 akun terdaftar.
11
Performance Testing
Perf Test
 Non-Functional Pengujian untuk mengukur kecepatan, stabilitas, dan skalabilitas sistem di bawah beban kerja tertentu. Memastikan sistem merespons cepat dan stabil sesuai dengan standar performa yang ditetapkan.
Contoh
Halaman daftar produk harus load dalam <2 detik dengan 100 produk — QA mengukur dengan tools dan melaporkan jika melebihi threshold.
12
Load Testing
Load Test
 Non-Functional Pengujian perilaku sistem saat menerima beban user yang diharapkan dalam kondisi normal hingga puncak. Mengetahui batas kapasitas sistem dan memastikan sistem stabil di traffic normal maupun peak.
Contoh
Simulasi 5.000 user mengakses aplikasi e-commerce bersamaan saat flash sale — QA memantau response time, error rate, dan CPU usage server.
13
Stress Testing
Stress Test
 Non-Functional Pengujian sistem di luar batas kapasitas normal untuk melihat bagaimana sistem berperilaku dan apakah bisa pulih. Mengetahui titik kegagalan sistem (breaking point) dan kemampuan recovery-nya.
Contoh
Traffic dinaikkan terus hingga server mulai error — QA mencatat di angka berapa concurrent user sistem mulai crash dan apakah aplikasi bisa restart sendiri.
14
Security Testing
Security / Pentest
 Non-Functional Pengujian untuk mengidentifikasi celah keamanan, vulnerability, dan potensi ancaman pada sistem. Melindungi data user dan sistem dari serangan seperti SQL injection, XSS, dan unauthorized access.
Contoh
QA mencoba memasukkan ' OR '1'='1 di field login — sistem yang aman harus menolak input tersebut dan tidak memberikan akses.
15
Usability Testing
UX Test
 Non-Functional Pengujian dengan melibatkan user nyata untuk mengevaluasi seberapa mudah dan intuitif sistem digunakan. Mengidentifikasi hambatan UX yang membuat user kebingungan atau frustasi saat menggunakan aplikasi.
Contoh
5 orang diminta menyelesaikan tugas "beli produk dan checkout" tanpa panduan — QA mengamati di mana mereka berhenti atau melakukan kesalahan, lalu melaporkan hambatan UX.
16
Compatibility Testing
Cross-platform Test
 Non-Functional Pengujian apakah aplikasi berjalan dengan benar di berbagai browser, OS, device, dan resolusi layar. Memastikan pengalaman user konsisten di semua platform yang didukung.
Contoh
Halaman checkout ditest di Chrome, Firefox, Safari, dan Samsung Internet — ditemukan bug: tombol "Bayar" tidak muncul di Safari iOS karena masalah CSS.
17
Accessibility Testing
a11y Test
 Non-Functional Pengujian apakah aplikasi dapat digunakan oleh semua orang, termasuk penyandang disabilitas (tunanetra, tunarungu, dll). Memastikan aplikasi memenuhi standar aksesibilitas (WCAG) dan inklusif untuk semua user.
Contoh
QA menggunakan screen reader (NVDA) untuk menavigasi halaman login — memverifikasi bahwa setiap form field memiliki label yang dapat dibaca oleh screen reader.
18
White Box Testing
Glass Box / Clear Box
 Structural Pengujian yang dilakukan dengan pengetahuan penuh tentang struktur internal kode — tester bisa melihat source code. Memastikan semua jalur logika kode (code path, branch, condition) sudah dieksekusi dan ditest.
Contoh
Developer menulis unit test yang mengcover semua branch dalam fungsi validasi — termasuk kondisi if/else dan edge case nilai null atau kosong.
19
Black Box Testing
Behavioral Test
 Structural Pengujian tanpa pengetahuan tentang internal kode — tester hanya mengetahui input dan output yang diharapkan. Memverifikasi behavior sistem dari perspektif user, berdasarkan requirement dan spesifikasi.
Contoh
QA mentest fitur transfer bank tanpa tahu cara kode backend bekerja — hanya memasukkan nominal dan memverifikasi apakah saldo berkurang dan penerima menerima uang.
20
Regression Testing
Regression Test
 Regression Pengujian ulang terhadap fitur-fitur yang sudah ada setelah terjadi perubahan kode untuk memastikan tidak ada yang rusak. Mencegah bug baru muncul di area yang tidak berkaitan dengan perubahan yang dilakukan.
Contoh
Developer update fitur pencarian produk — QA menjalankan regression test pada fitur cart, checkout, dan login untuk memastikan perubahan tersebut tidak membreak fitur lain.
21
End-to-End Testing
E2E Test
 Regression Pengujian keseluruhan alur bisnis dari awal hingga akhir, mensimulasikan pengalaman nyata seorang user. Memastikan seluruh alur bisnis berjalan dengan benar dari perspektif user secara end-to-end.
Contoh
Test otomatis menjalankan skenario: buka aplikasi → register akun baru → verifikasi email → login → order produk → bayar → cek status order — semua step harus berhasil.
22
Retesting
Bug Verification
 Maintenance Pengujian ulang yang dilakukan khusus untuk memverifikasi bahwa bug yang dilaporkan sebelumnya sudah benar-benar diperbaiki. Memastikan fix yang dilakukan developer sudah menyelesaikan bug sesuai yang dilaporkan.
Contoh
Bug BUG-101 "harga produk tampil Rp0 di halaman detail" sudah difix — QA menjalankan ulang langkah reproduksi asli dan memverifikasi harga kini tampil dengan benar.

Catatan: Beberapa tipe testing dapat tumpang tindih kategori tergantung konteks penggunaannya. Misalnya, E2E Testing bisa bersifat functional sekaligus regression. Kategorisasi di atas mengikuti pendekatan yang paling umum digunakan di industri.
